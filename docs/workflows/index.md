# Workflows

Step-by-step reference guides that define **how BSE employees carry out recurring processes**. Each workflow covers both the human steps (what you do) and the system steps (what happens automatically), so that as our systems evolve we keep their data synced and every new joiner has a clear path to follow.

## Available workflows

| Workflow | Purpose |
|---|---|
| [Managing Website Content](managing-website-content.md) | Keep bse.com.eg up to date — publish news, update products, manage partners, handle translations, and more. |
| [Daily Commit Summary](daily-commit-summary.md) | Generate a summary of all commits pushed to BSE repos for a given day or week. |
| [WFH Policy](wfh-policy.md) | Four-step process for working from home: request to HR → approval → morning check-in → daily report before 6 PM. |

## How to use these guides

1. **Find the workflow** that matches what you need to do.
2. **Follow the steps** in order — each one tells you who acts (human or system) and what the expected outcome is.
3. **Check the screenshots** for visual confirmation you're in the right place.
4. **Use the quick-reference tables** at the top of each workflow to jump to the section you need.

## Adding a new workflow

Create a new `.md` file in this directory. It will appear in the navigation automatically — no configuration changes needed.
